1. async function (비동기 함수) : 총 5개인터넷을 거쳐 Supabase 데이터베이스(서버)와 데이터를 주고받는 통신 전용 함수들입니다. 서버 응답을 기다려야 하므로 전부 async가 붙어 있습니다.  handleAuthSubmit() : 로그인 및 새 팀방 개설 처리   fetchAllDataFromServer() : 서버에서 전체 베드 상태 및 설정을 안전하게 가져오기   checkAutoLogin() : 브라우저에 저장된 ID로 자동 로그인 확인   saveBedStateToServer(roomNum) : 특정 치료방의 상태 변화를 실시간으로 서버에 저장   deleteHistoryItem(index) : 완료된 치료 내역을 서버에서 삭제하고 치료사 카운트 차감   2. 일반 function (함수) : 약 30여 개서버 통신 없이 내 컴퓨터 브라우저 안에서 화면을 바꾸거나, 버튼을 누르면 즉시 계산해서 UI를 그려주는 함수들입니다. 역할별로 묶어보면 다음과 같습니다.로그인 & 로그아웃 관리 (3개)updateLastKnownCache(), switchAuthTab(), handleLogout()   설정 및 환자 선택 모달창 제어 (9개)openQuickTreatment(), closeQuickTreatment(), applyQuickTreatment()   toggleModal(), renderModalStaffList(), addStaffConfigField()   saveGlobalSettings(), renderModalRoutineList(), addRoutineRowField()   완료된 치료 기록(히스토리) 관리 (4개)loadMoreHistory(), searchHistory(), renderHistoryLogTable(), clearHistoryLog()   치료방(베드) 및 타이머 제어 (14개 이상)toggleRoomEdit(), toggleNextSchedule(), addTemporaryTreatment()   refreshBuilderButtons(), clickBuilderItem(), applyCustomRoutine()   resetBuilder(), forceResetRoom(), autoAssignStaffAndNameByRange()   resumeTimerLoop(), updateRoomName(), quickEditRoomName(), updateRoomStaff(), initDashboard(), playBeep() 등 (화면 갱신 및 타이머 구동용 필수 함수들)   3. 전역 let (변수 상자) : 총 14개코드 최상단에서 프로그램이 실행되는 동안 전체 화면과 서버가 공유해야 하는 핵심 기억 상자들입니다. 수시로 데이터가 바뀌기 때문에 const가 아닌 let으로 선언되어 있습니다.  로그인 정보: currentTeamId, currentTeamName, authMode   조회용 정보: historySearchKeyword, historyDisplayCount, historyExpanded, currentQuickRoom   실시간 데이터의 핵심:config : 총 방 개수, 치료사 목록, 기본 치료 루틴 정보   roomStates : 현재 각 치료방의 타이머, 진행 단계, 배정된 치료사 상태   historyLog : 오늘 완료된 치료 누적 목록   무한 루프 방지용 가드: isInternalUpdating, lastKnownRoomStates, lastKnownConfig, lastKnownHistoryLog   (※ 이외에 함수 내부에서 let state, let index 처럼 임시로 잠깐 계산하고 버리는 지역 변수들도 수십 개 사용되고 있습니다.)
